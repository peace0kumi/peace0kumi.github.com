선언

CSS 선언은 external방식을 권장한다.

<link rel="stylesheet" type="text/css" href="category.css" />

charset

external방식의 문서 charset은 utf-8을 기본으로 하며, HTML과 동일한 인코딩을 문서 첫 줄에 공백 없이 선언한다.

파일을 저장할 때는 반드시 선언한 인코딩과 동일한 인코딩을 선택한다.

@charset "utf-8";

초기화

module로 초기화 하며, element의 user agent stylesheet 속성은 초기화 하지 않는다.

표기법

기본 형식

속성 간 개행하지 않으며, 선택자와 속성 및 속성값은 한 칸씩 공백을 둔다. 단 중괄호 '{}' 가 중첩되는 경우 예외로 한다.

.selector {property: value; property: value;}

@media all and (min-width: value;) {
	.selector {property: value; property: value;}
}

선택자

선택자는 class를 기본으로 한다. 태그네임을 선택자로 사용하는 경우는 user agent stylesheet를 초기화할 경우만 허용한다.

.selector {font-size: 1em; line-height: 1.5; color: #000000;}
p.selector {margin: 0;}
span.selector {display: block;}

주석 표기

주석은 선택자와 동일 라인에 표기 하지 않으며, 분류와 모듈명을 반드시 기입한다.

수정 및 추가작업 후에는 날짜와 함께 수정내역을 기입한다.

/*
 * module name
 */
.module {property: value; property: value;}

/* module type 2016-01-01 수정 내용 */
.module.type {property: value; property: value;}

네이밍

선택자

약어 사용을 지양하며, 네이밍의 시작은 반드시 영문 소문자로 시작한다. 단어별 구분은 id의 경우 카멜 표기법(Camel Notation)으로 class의 경우 hyphen으로 표기한다.

#myPage {property: value; property: value;}
.my-page {property: value; property: value;}

규약

분류 정의 예시
module 레이아웃과 관계없이 단독으로 사용 가능한 UIO em, form, btn, data-list, data-table, tabs... 등의 약속어가 존재
theme theme-a, theme-b, theme-c... 로 확장
type 테마 내의 디자인 분류 type-a, type-b, type-c... 로 확장
outline 모듈이 위치할 문서구조 단위 x-small, small, medium, large, x-large 로 확장
level 상황적 의미 또는 정보의 중요도 … fall-03, fall-02, fall-01, rise-01, rise-02, rise-03... 로 확장
symbols 픽토그램 등의 시각적 표현이 필요할 경우 before-*, after-*
function 상호작용의 기능 및 상태 active, current...
