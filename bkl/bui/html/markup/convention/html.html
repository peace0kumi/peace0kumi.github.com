title

각 페이지마다 서로 다른 규칙으로 사용 중인 문서 제목 (<title> 태그) 규칙을 일원화함으로써, 서비스 통일성을 높이고 문서 제목에 콘텐츠 제목을 포함시킴으로써 검색 엔진에 대한 최적화를 꾀한다.

<!-- 기본 규칙 -->
<title>사이트명 서비스명</title>

<!-- 하위 컨텐츠 -->
<title>컨텐츠명 - 사이트명 서비스명</title>

표기법

기본 형식

  1. 요소의 완벽한 중첩 되어야 한다.
  2. 모든 속성 값은 속성이 함께 선언되어야 한다.
  3. 요소와 속성은 소문자여야 한다.
  4. 속성 값은 인용 부호("") 안에 표기한다.
  5. 부위 인식자의 모든 NAME속성을 ID 속성으로 대체한다.
  6. 모든 요소는 닫아야 한다.
  7. text나 URL, script에 포함된 특수 문자는 escape 시킨다.
  8. 블록 요소와 인라인 요소를 형제로 놓지 않는다.

속성(atribute) 선언 순서

  1. rel
  2. type
  3. href, src
  4. width, height
  5. target
  6. name
  7. Name, alt
  8. 기타
  9. id
  10. class
  11. style

들여 쓰기

코드의 가독성을 높이기 위하여 들여 쓰기를 사용하며 탭 1개의 크기는 공백 4칸으로 설정한다. 또한 마크업의 깊이가 깊어질 때마다 탭 1개만큼 들여 쓰지만 다음의 경우 들여 쓰지 않을 수 있다.

자식 요소가 단독으로 쓰일 수 없으며, 또 다른 자식 요소를 포함할 수 있는 경우 thead, tfoot, tbody, tr, li, dt, dd 단(th, td는 제외)

<!-- thead, tfoot, tbody, tr -->
<table>
<tbody>
<tr>
	<td></td>
	<td></td>
</tr>
</tbody>
</table>
<!-- //thead, tfoot, tbody, tr -->

<!-- li -->
<ul>
<li></li>
<li></li>
</ul>
<!-- //li -->

<!-- dt, dd -->
<dl>
<dt></dt>
<dd></dd>
</dl>
<!-- //dt, dd -->

주석 표기

마크업 수정 시, 수정 부분을 표기하는 주석은 다음과 같이 날짜를 기입하며, element와 동일 라인에 표기하지 않는다.

<!-- uio-name 2016-01-01 수정 내용 -->
<element>
	<element>
	</element>
</element>
<!-- //uio-name 2016-01-01 수정 내용 -->

빈 줄

빈 줄은 존재하지 않는 것을 원칙으로 한다. 단, 콘텐츠의 구분을 위하여 부득이하게 사용해야 할 경우 1줄을 초과하지 않는다.

<!-- uio-name -->
<element>
	<element>
	</element>
</element>
<!-- //2016-01-01 -->

<!-- uio-name -->
<element>
	<element>
	</element>
</element>
<!-- //uio-name -->

구조

기본 구조

섹션의 시작은 반드시 제목을 표기하여 문서의 흐름이 끊기지 않게 한다.

<!-- latest-posts -->
<div class="latest-posts">
	<hx>notice</hx>
	<ul>
	<li>Notice is the latest posts.</li>
	<li>Notice is the latest posts.</li>
	</ul>
	<div class="aside"><a href=”#”>more</a></div>
</div>
<!-- //latest-posts -->

네이밍

기본규칙

약어 사용을 지양하며, 네이밍의 시작은 반드시 영문 소문자로 시작한다. 단어별 구분은 id의 경우 카멜 표기법(Camel Notation)으로 class의 경우 hyphen으로 표기한다.

<element class="my-page" id="myPage"></element>
